# Help Center Category Management

Admin UI for creating, editing, deleting, and reordering help center categories.

## Problem

The backend fully supports category CRUD (create, update, delete, list) with mutation hooks wired up, but the admin UI has no way to manage categories. They only appear as a dropdown when editing articles. Admins cannot create new categories, rename existing ones, reorder them, or control their visibility.

## Scope

**In scope:**

- CategoryNav sidebar component on `/admin/help-center`
- CategoryFormDialog (create/edit) with name, description, emoji icon, public/private toggle
- Drag-to-reorder via `position` field
- Inline category creation from article editor's category dropdown
- Status filter relocation from sidebar to article list header

**Out of scope:**

- Nested/hierarchical categories (schema supports `parentId`, deferred to later)
- Full emoji picker library (simple custom popover instead)
- Backend changes (all CRUD operations already exist)
- TipTap editor changes (already at feature parity with posts/changelog)

## Architecture

### 1. CategoryNav Sidebar

Replaces `HelpCenterFiltersPanel` in the existing `AdminFilterLayout` left sidebar. Follows the `UsersSegmentNav` pattern from the users section.

**Structure:**

- Header: "Categories" label + "+" create button
- "All Articles" row: always first, shows total count, clears category filter
- Divider
- Category list: each item shows emoji icon, name, article count
  - Click: filters articles to that category (URL param `?category=<id>`)
  - Hover: reveals edit (pencil) + delete (trash) actions via CSS `group-hover`
  - Drag handle on left for reorder

**Selection state:** URL search param `?category=<id>` on the help center route. Single-select only (click toggles, click "All Articles" clears).

**Files to create:**

- `apps/web/src/components/admin/help-center/help-center-category-nav.tsx`

**Files to modify:**

- `apps/web/src/components/admin/help-center/help-center-list.tsx` вЂ” swap `HelpCenterFiltersPanel` for `CategoryNav`, move status filter inline
- `apps/web/src/routes/admin/help-center.tsx` вЂ” add `category` search param to route

### 2. CategoryFormDialog

Shared dialog for create and edit, opened from:

- Sidebar "+" button (create mode)
- Sidebar hover edit action (edit mode, pre-filled)
- Article editor category dropdown "+" button (create mode, auto-selects on success)

**Fields:**

- **Emoji icon**: clickable button showing current emoji. Opens a Popover (shadcn `Popover` + `PopoverContent`) with a grid of ~50 common category emojis. Same pattern as the reaction emoji picker in `widget-comment-list.tsx` and `comment-thread.tsx`, but with a `CATEGORY_EMOJIS` constant and a wrapped grid layout instead of a single row. Stored in the `icon` text field as a unicode character. Default: `рџ“Ѓ`. If `icon` is null/empty, display `рџ“Ѓ` as fallback.
- **Name**: text input, required. Slug auto-generated by backend.
- **Description**: optional textarea, 1-2 lines.
- **Public toggle**: switch component, default on. When off, category hidden from public help center.

**State management:** local `useState` for form fields. `useEffect` resets form when dialog opens with `initialValues` (edit mode).

Uses existing `useCreateCategory` / `useUpdateCategory` mutation hooks. On success, invalidates `helpCenterQueries.categories()`.

**Files to create:**

- `apps/web/src/components/admin/help-center/category-form-dialog.tsx` вЂ” includes emoji picker inline (same Popover pattern as comment reactions, no separate component needed)

### 3. Drag-to-Reorder

Uses `@dnd-kit/core` + `@dnd-kit/sortable` (already installed).

- Drag handle (grip dots icon) on each category item
- On drag end: compute new `position` values, optimistic cache update, batch `updateCategoryFn` calls
- Rollback on error

Integrated directly into the `CategoryNav` component.

### 4. Inline Category Creation from Article Editor

Add a "+" button next to the category `<select>` in `help-center-metadata-sidebar.tsx`. Opens `CategoryFormDialog` in create mode. On success, auto-selects the newly created category in the dropdown.

**Files to modify:**

- `apps/web/src/components/admin/help-center/help-center-metadata-sidebar.tsx`

### 5. Status Filter Relocation

The status filter (All/Published/Draft) currently lives in `HelpCenterFiltersPanel` in the sidebar. Since the sidebar becomes the category nav, move the status filter inline into the `AdminListHeader` area above the article list.

**Implementation:** Add a small segmented control or dropdown next to the search input in the article list header.

**Files to modify:**

- `apps/web/src/components/admin/help-center/help-center-list.tsx`

### 6. Delete Safety

When deleting a category that has articles (`articleCount > 0` from the categories list query):

- Confirmation dialog shows: "This category has N articles. Reassign or delete them first."
- Confirm button is disabled
- Category can only be deleted when empty

Uses existing `ConfirmDialog` component.

## Existing Backend (no changes needed)

- **Server functions:** `createCategoryFn`, `updateCategoryFn`, `deleteCategoryFn`, `listCategoriesFn` in `lib/server/functions/help-center.ts`
- **Service layer:** Full CRUD with slug generation, soft delete, position ordering in `lib/server/domains/help-center/help-center.service.ts`
- **Mutation hooks:** `useCreateCategory`, `useUpdateCategory`, `useDeleteCategory` in `lib/client/mutations/help-center.ts`
- **Query hooks:** `helpCenterQueries.categories()` in `lib/client/queries/help-center.ts`
- **Schema:** `helpCenterCategories` table with `name`, `slug`, `description`, `icon`, `isPublic`, `position`, `parentId` in `packages/db/src/schema/kb.ts`
